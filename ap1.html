AP 1ère Année

Situations professionnelles réalisées en première année de BTS SIO SISR : installation de GLPI et mise en place d'un serveur Web.

Installation GLPI
Contexte

Dans cet atelier de professionnalisation, nous avons appris à installer et à utiliser GLPI sur un serveur GNU/Linux, plus particulièrement Ubuntu. Afin de nous initier au système de ticketing qui est très utilisé dans le monde de l'entreprise.

Objectifs

Chaque entreprise dispose d'un parc informatique plus ou moins conséquent. Ce parc nécessite une gestion (inventaire, tâches administratives…) à distance ainsi que d'une plateforme d'assistance fonctionnant sur la base de signalement d'incident. Dans le cadre de cette gestion, mise en place d'un logiciel libre de gestion de parc informatique : GLPI.

Environnement technologique

Le logiciel GLPI sera installé sur une machine virtuelle Ubuntu 16. Le logiciel GLPI fonctionne avec une base de données ainsi qu'un serveur WEB (gestion par application WEB) : utilisation des services complémentaires : un serveur WEB Apache ; un serveur de base de données MySQL et un interpréteur PHP.

Réalisation
  • Mise à jour des ressources paquets de la machine
  • Installation des services complémentaires (Apache, MySQL, PHP)
  • Installation de GLPI
  • Mise en service du logiciel via navigateur WEB (création utilisateur et parc informatique)
  • Gestion des tickets d'incidents

Installation Serveur Web
Contexte

Dans le cadre d'AP Support & Système lors de la 1ère année de BTS SIO option SISR : installation et mise en service d'un serveur HTTP à destination d'une entreprise.

Objectifs

Une entreprise disposant d'un service en ligne (site vitrine, application WEB…) dépend d'un serveur HTTP permettant l'offre de service ainsi que la mise à jour des informations. Ce serveur doit être disponible à distance afin que les collaborateurs puissent le modifier à tout moment. Dans le cadre de ce service : installation et mise en service d'un serveur HTTP.

Environnement technologique

Le serveur HTTP sera une machine virtuelle Debian-Server 10. L'accès au serveur se fera via une machine virtuelle Debian-Client 10. Le fonctionnement du serveur nécessite différents outils : un serveur WEB Apache ; un serveur de base de données MariaDB avec l'interface phpMyAdmin ; un interpréteur PHP. La gestion à distance des applications disponibles sur le serveur HTTP nécessite un outil de transfert de fichiers : serveur FTP ProFTPD.

Debian Server 10 (VM serveur)
Debian Client 10 (VM client)
Apache
MariaDB + phpMyAdmin

Réalisation

La mise en place de serveurs dans le cadre d'une entreprise est un point essentiel. Chaque collaborateur doit être capable d'accéder aux applications des différents serveurs de manière fiable et sécurisée afin de garantir l'efficacité des services proposés. Je pense que pouvoir proposer ce genre de services aux clients est essentiel pour un professionnel.
